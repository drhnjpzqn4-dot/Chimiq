## 2. Rebuild + ship the iOS app — makes #2/#4/flag/#3-client live on phone

```bash
cd ~/PiasVentures/chimiq-code \
  && rm -rf artifacts/skinscreen/dist \
            artifacts/skinscreen/mobile/capacitor/ios/App/App/public \
  && pnpm build:mobile \
  && cd artifacts/skinscreen/mobile/capacitor \
  && npx cap open ios
```

In Xcode:
1. Set the run target to **Any iOS Device (arm64)** (not a simulator, or Archive is greyed out).
2. **Product → Archive**, wait for the Organizer to open.
3. **Distribute App → TestFlight & App Store Connect → Upload** (automatic signing).
4. If it says the build number is taken: target **App → General → Build**, +1, re-Archive.
5. Do **not** set `CAP_SERVER_URL` — this must be a bundled build.

## 4. Refill the catalog with REAL ingredient data (on Stina / Mac mini)

The old Lyko harvest captured page navigation + reviews instead of INCI. `harvest_lyko.py`
is now hardened to store a clean INCI list **or nothing**. Copy the new file up first:

```bash
scp ~/PiasVentures/chimiq-code/harvest_lyko.py   ai_stina@Pias-Mac-mini.local:~/harvest/
scp ~/PiasVentures/chimiq-code/harvest_apotea.py ai_stina@Pias-Mac-mini.local:~/harvest/
ssh ai_stina@Pias-Mac-mini.local
cd ~/harvest && source ~/.zprofile
```

**Apotea = primary source (reliable INCI).** Run real batches:

```bash
caffeinate -i python3 harvest_apotea.py --category serum            --limit 200
caffeinate -i python3 harvest_apotea.py --category dagkram          --limit 150
caffeinate -i python3 harvest_apotea.py --category nattkram         --limit 150
caffeinate -i python3 harvest_apotea.py --category ansiktsrengoring --limit 150
caffeinate -i python3 harvest_apotea.py --category toner            --limit 100
caffeinate -i python3 harvest_apotea.py --category solskydd         --limit 100
```

**Lyko = test small first** to see whether the hardened extractor actually finds INCI
(watch the per-product `INCI:✅ / –` column). If most rows show `INCI:–`, Lyko doesn't
expose the list in the page → rely on Apotea + OBF-by-EAN instead, don't run big Lyko batches:

```bash
caffeinate -i python3 harvest_lyko.py --category serum --limit 10
```

Data lands in `scraped_products` (staging). To promote clean rows into the live
`cached_products`, only rows with a real barcode + a validated INCI list should move
across (see SS-079 promotion logic). Ask Claude to run the promote when a batch finishes.
