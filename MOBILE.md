# SkinScreen native build (iOS + Android)

The mobile shells live in [`mobile/capacitor`](./mobile/capacitor) and wrap the **`artifacts/skinscreen`** PWA as native iOS and Android apps using Capacitor 6, in a **reader-app pattern**: all subscriptions are sold and managed on the web; the native shells only read entitlement.

This file is the quick-start. The full build instructions, plugin list, asset regeneration commands, and store-submission checklists live in:

* [`mobile/capacitor/README.md`](./mobile/capacitor/README.md) — daily build flow
* [`artifacts/skinscreen/store/listing-en.md`](./artifacts/skinscreen/store/listing-en.md) — English storefront copy
* [`artifacts/skinscreen/store/listing-sv.md`](./artifacts/skinscreen/store/listing-sv.md) — Swedish storefront copy
* [`artifacts/skinscreen/store/privacy.md`](./artifacts/skinscreen/store/privacy.md) — App Privacy / Data Safety answers
* [`artifacts/skinscreen/store/reviewer-notes.md`](./artifacts/skinscreen/store/reviewer-notes.md) — App Review notes (with demo account)

## Why a wrapper, not React Native / Expo?

* Web is the source of truth — every feature ships once.
* Reader-app compliance is unambiguous: payments live on the web only.
* Crowdsourced ingredient DB updates instantly without store review.
* The barcode scanner, OCR, and AI chat all work in mobile Safari / Chrome already; we just need a native shell so users can install from the stores.

## Why this folder isn't part of the pnpm workspace

The Replit Linux container can't run Xcode or Android Studio, so `mobile/capacitor` lives **outside** `pnpm-workspace.yaml`. Bootstrap it on your dev machine:

```bash
cd mobile/capacitor
npm install
npx cap add ios       # macOS only
npx cap add android   # any OS, needs JDK 17 + Android Studio
```

Both `cap add` commands generate the platform-specific projects. They are intentionally not committed.

---

## Daily flow

```bash
# Repo root
pnpm build:mobile

# Then
cd mobile/capacitor
npm run sync          # copies dist/ + capacitor.config into ios/android
npm run open:ios      # → Xcode
npm run open:android  # → Android Studio
```

Override the served URL at sync time:

```bash
CAP_SERVER_URL=https://staging.skinscreen.chimiq.com npm run sync
```

## What is **not** in this commit (deferred to your dev machine)

* The actual `mobile/capacitor/ios/` and `mobile/capacitor/android/` projects — they require Xcode / Android Studio to bootstrap. Run `npx cap add ios|android` once locally and they'll appear.
* Per-density icon and splash variants — generated by `npx @capacitor/assets generate` from the masters in `store/`.
* Real device screenshots for the storefront — capture from the demo account on Simulator / Emulator (see `store/screenshots/README.md`).
* Code signing certificates and `.aab` / `.ipa` artefacts — produced inside Xcode / `gradle bundleRelease`.

These steps were intentionally deferred because the hosting environment cannot meaningfully execute or validate them. Everything required to start them is present in this repo.
