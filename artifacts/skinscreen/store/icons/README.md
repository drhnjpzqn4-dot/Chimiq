# Icon master assets

| File | Use |
| --- | --- |
| `icon-1024.png` | App Store / Play Store hero icon. 1024×1024, no transparency, no rounded corners (the OS adds the mask). |
| `icon-1024-maskable.png` | PWA + Android adaptive icon foreground source. Safe-zone is the inner 80%. |
| `ios-app-180.png` | iOS home-screen 60pt @3x — sanity-check render. The actual sizes are generated by `npx @capacitor/assets`. |
| `android-foreground-432.png` | Android adaptive-icon foreground (432×432). Background colour: `#FAF7F2` (light) / `#1F2937` (dark). |

## Regenerating

From the repo root:

```bash
magick artifacts/skinscreen/public/icon-512.png -resize 1024x1024 \
  artifacts/skinscreen/store/icons/icon-1024.png
```

Then have Capacitor blow them out into per-density variants. Run from `mobile/capacitor` so the relative `--assetPath` resolves correctly:

```bash
cd mobile/capacitor
npx @capacitor/assets generate \
  --iconBackgroundColor '#FAF7F2' \
  --iconBackgroundColorDark '#1F2937' \
  --assetPath ../../artifacts/skinscreen/store
```

This produces:

* iOS: `ios/App/App/Assets.xcassets/AppIcon.appiconset/*`
* Android: `android/app/src/main/res/mipmap-*/`
* Web (PWA): refreshes the existing `public/icon-*.png` (already in repo).
